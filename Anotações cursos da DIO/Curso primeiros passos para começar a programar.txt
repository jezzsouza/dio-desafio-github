Tema 3 - Algoritmo

O computador tem como objetivo  o processamento de dados.
Algoritmos são um processo de resolução de problemas "atep by step"(passo a passo) utilizando instruções.

Detalhamento para o PC
instruções:
O que precisa ser feito?
Qual a ordem de execução?

Desenvolvimento do programa

Instruções 
Análise: estudo e definição dos dados de entrada e saída.

Algoritmo
Uso: descreve o problema por meio de ferramentas narrativas - simples entendimento - , fluxogramas - utilização de símbolos pré-definidos - ou pseudocódigo - estrutura e símbolo .

Programa
Codificação: o algoritmo é codificado de acordo com a linguagem de programação escolhida

Como construir um algoritmo?

Compreensão do problema - pontos mais importantes;
Definição dados de entrada - dados fornecidos e cenário;
Definir processamento - cálculos e restrições;
Definir dados de saída - após processamento;
Utilizar um método de construção - construção e refinação do algoritmo;
Teste e diagnóstico.

Binário - precisa de duas variáveis ou uma variável e uma constante, ou seja, precisa de dois objetos para executar a operação. 

Unário - utilizado para inversão de sinal

Estrutura condicional

Condição - estado de uma pessoa ou coisa.
Condicional - que expressão, uma condição ou suposição;
                        contém ou implica uma suposição ou hipótese.

Simples - executa;
Composta - contém uma condição;
encadeada - mais de uma condição.

Estrutura de repetição

Enquanto....faça;
Repita.....até;
Para.....de.....até.....faça

Entrada/saída

Existem dois tipos de saídas dentro de um programa (algoritmo)
Saída por interrupção: definida pelos periféricos. Interrompendo a execução do programa e baixando os resultados.

Saída programada: 
1. Condicional - aguada que o dispositivo esteja preparado para receber a informação
2. Incondicional - insere os dados independente do estado do dispositivo.

Como o computador entende o programa

Tradução:  
1. Geração do programa objeto;
2.Execução do programa objeto.

A linguagem que traduzida é executada mais rápido.

Interpretação:
1. Programa fonte executado diretamente.

A linguagem que interpretada tem programas menores e maior flexibilidade.
